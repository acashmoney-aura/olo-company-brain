# Planner Checklist

- follow repo contract
- stay inside allowed paths
- compare desired state vs current state before acting
- read the latest project audit and current blocker before advancing work
- keep plans, assumptions, decisions, and blockers in markdown under research_plan
- use role checklists as the approval contract before creating or launching worker tasks
- require ontology-backed source configs for required datasets
- flag placeholder, estimated, synthetic, or missing data explicitly
- do not advance a task until verification, publish status, and audited repo state agree
- launch the smallest next worker task and then re-plan from updated repo state
- satisfy deterministic completion checks
